# Internacionalización (i18n) - Guía de Uso

Este proyecto utiliza el sistema de enrutamiento i18n integrado de Astro para soportar múltiples idiomas.

## 📖 Configuración Actual

### Idiomas Soportados

- **Inglés (en)**: Idioma por defecto
- **Español (es)**: Idioma secundario

### Estructura de URLs

```
example.com/          → Inglés (sin prefijo)
example.com/about/    → Inglés (sin prefijo)

example.com/es/       → Español
example.com/es/about/ → Español
```

### Configuración en `astro.config.mjs`

```js
i18n: {
  defaultLocale: "en",
  locales: ["en", "es"],
  fallback: {
    es: "en",  // Si falta contenido en español, mostrar en inglés
  },
  routing: {
    prefixDefaultLocale: false,  // No usar /en/ para inglés
    fallbackType: "rewrite",     // Reescribir en lugar de redirigir
  },
}
```

## 📁 Estructura de Archivos

Para crear páginas multiidioma:

```
src/pages/
├── index.astro           # Página en inglés (/)
├── about.astro           # Página en inglés (/about)
└── es/
    ├── index.astro       # Página en español (/es/)
    └── about.astro       # Página en español (/es/about)
```

## 🔧 Funciones de Utilidad

Importa las funciones helper de `@/utils/i18n`:

```astro
---
import { getLocalizedPath, getCurrentLocale } from "@/utils/i18n";

// Obtener URL localizada
const aboutUrl = getLocalizedPath("es", "about"); // => "/es/about"

// Obtener locale actual
const locale = getCurrentLocale(Astro.currentLocale); // => "en" | "es"
---
```

### Funciones Disponibles

#### `getLocalizedPath(locale, path)`
Obtiene la ruta relativa para un idioma específico.

```ts
getLocalizedPath("es", "about");  // => "/es/about"
getLocalizedPath("en", "about");  // => "/about"
```

#### `getAbsoluteLocalizedPath(locale, path)`
Obtiene la URL absoluta para un idioma específico.

```ts
getAbsoluteLocalizedPath("es", "about");
// => "https://example.com/es/about"
```

#### `getAllLocalizedPaths(path)`
Obtiene todas las URLs relativas de una ruta en todos los idiomas.

```ts
getAllLocalizedPaths("about");
// => ["/about", "/es/about"]
```

#### `getCurrentLocale(astroLocale)`
Obtiene el locale actual con validación y fallback.

```ts
const locale = getCurrentLocale(Astro.currentLocale); // => "en" | "es"
```

#### `getAlternateLocale(currentLocale)`
Obtiene el idioma alternativo (para selector de idioma).

```ts
getAlternateLocale("en"); // => "es"
getAlternateLocale("es"); // => "en"
```

## 🎯 Uso en Componentes

### Ejemplo: Selector de Idioma

```astro
---
import { getLocalizedPath, getAlternateLocale, localeLabels } from "@/utils/i18n";

const currentLocale = Astro.currentLocale || "en";
const alternateLocale = getAlternateLocale(currentLocale);
const currentPath = Astro.url.pathname.replace(/^\/(es\/)?/, "");
const alternateUrl = getLocalizedPath(alternateLocale, currentPath);
---

<a href={alternateUrl}>
  {localeLabels[alternateLocale]}
</a>
```

### Ejemplo: Links Internos

```astro
---
import { getRelativeLocaleUrl } from "astro:i18n";

// Usar funciones nativas de Astro también funciona
const currentLocale = Astro.currentLocale || "en";
---

<nav>
  <a href={getRelativeLocaleUrl(currentLocale, "")}>Home</a>
  <a href={getRelativeLocaleUrl(currentLocale, "about")}>About</a>
  <a href={getRelativeLocaleUrl(currentLocale, "contact")}>Contact</a>
</nav>
```

### Ejemplo: Meta Tags Alternativos

```astro
---
import { getAllAbsoluteLocalizedPaths } from "@/utils/i18n";

const currentPath = Astro.url.pathname.replace(/^\/(es\/)?/, "");
const alternateUrls = getAllAbsoluteLocalizedPaths(currentPath);
---

<head>
  {alternateUrls.map((url, index) => {
    const locale = index === 0 ? "en" : "es";
    return <link rel="alternate" hreflang={locale} href={url} />;
  })}
</head>
```

## 🌐 Layout con i18n

El componente `Layout.astro` ya detecta automáticamente el idioma:

```astro
---
// src/layouts/Layout.astro
const lang = Astro.currentLocale || "en";
---

<html lang={lang}>
  <!-- ... -->
</html>
```

## 📝 Buenas Prácticas

### 1. Mantener Consistencia de Contenido

Asegúrate de crear versiones equivalentes en todos los idiomas:

```
src/pages/about.astro        ✅ Existe
src/pages/es/about.astro     ✅ Existe

src/pages/services.astro     ✅ Existe
src/pages/es/services.astro  ❌ Falta → Se mostrará fallback en inglés
```

### 2. Usar Funciones Helper

Usa las funciones de `@/utils/i18n` en lugar de construir URLs manualmente:

```astro
<!-- ❌ No hacer esto -->
<a href={currentLocale === 'en' ? '/about' : '/es/about'}>About</a>

<!-- ✅ Hacer esto -->
<a href={getLocalizedPath(currentLocale, 'about')}>About</a>
```

### 3. Metadata Localizada

Personaliza los metadatos según el idioma:

```astro
---
const title = currentLocale === "en" 
  ? "About Us" 
  : "Acerca de Nosotros";
  
const description = currentLocale === "en"
  ? "Learn more about our company"
  : "Conoce más sobre nuestra empresa";
---

<Layout head={{ title, description }}>
  <!-- ... -->
</Layout>
```

### 4. Contenido Estructurado (JSON-LD)

Incluye `inLanguage` en los datos estructurados:

```js
const structuredData = {
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": title,
  "inLanguage": currentLocale, // "en" o "es"
  // ...
};
```

## 🔍 Middleware

El middleware de autenticación (`src/middleware.ts`) se ejecuta **antes** del middleware i18n de Astro, que se aplica automáticamente.

El orden de ejecución es:
1. Tu middleware personalizado (auth)
2. Middleware i18n de Astro (automático)
3. Renderizado de la página

## 🚀 Agregar un Nuevo Idioma

Para agregar un nuevo idioma (por ejemplo, francés):

1. **Actualizar `astro.config.mjs`**:
   ```js
   i18n: {
     locales: ["en", "es", "fr"],
     fallback: {
       es: "en",
       fr: "en",
     },
     // ...
   }
   ```

2. **Actualizar `src/utils/i18n.ts`**:
   ```ts
   export const locales = ["en", "es", "fr"] as const;
   
   export const localeLabels: Record<Locale, string> = {
     en: "English",
     es: "Español",
     fr: "Français",
   };
   ```

3. **Crear carpeta de páginas**:
   ```
   src/pages/fr/
   ├── index.astro
   ├── about.astro
   └── ...
   ```

4. **Traducir contenido** en los nuevos archivos.
